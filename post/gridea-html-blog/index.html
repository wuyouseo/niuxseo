Gridea搭建强大的静态博客

· 2019-08-15 · # Gridea

程序介绍 想必大家都听说过 Hexo，Jekyll 之类的静态博客程序，今天要介绍一款由国人 EryouHao 开发的静态博客程序 Gridea，这是一款简单，开源，强大的静态博客程序，博客使用的是 Markdown 的写法，配备了 Windows 和 Mac OS 的客户端，只需用手敲打文章就能一键生成同步到 Github 或 Coding。Gridea 暂时有四款主题，评论系统可以使用的是 Gitalk 或 Disqus，在客户端里打上相关信息就能实现一键配置，方便快捷。

上手实操 开始之前我们需要安装一个客户端，客户端在官网可以下载，Gridea 官网：https://gridea.dev 安装完成后我们能看到如下界面，这些便是博客内容的基本需求。

文章标签界面

然后试着新建一篇文章，在这里我们可以自定义 URL，增减文章标签，修改发布时间以及封面头图，并且还能统计字符单词的个数和自然段数，一目了然，希望能再增加一个定时保存文章草稿的功能就更好了！

可以在博客头部的菜单增加选项，不仅支持文章内的页面和标签链接，还可以输入外链。

配置评论

前面我说过 Gridea 支持 Gitalk 和 Disqus 两款评论套件二选一，总的来说各有各的优点，也各有各的缺点。

如果使用 Gitalk 需要在 Github 里面做一份 Gitalk 套进来才能使用，要是博客选择的是 Gitalk，那么评论的人需要注册使用 Github 账号才能评论。

如果使用 Disqus 需要在 Disqus 申请一个评论系统，要是博客选择的是 Disqus，那么评论系统很有可能会被墙掉。

上传博客 我这里选择的是 Coding Pages，Coding Pages 服务器在香港，速度会比 Github Pages 有优势。基本上填写好信息，可以点一下远程链接测试，没有问题保存即可，然后点击左下方的同步按钮，Gridea 就会生成网页并自动同步到 Coding Pages 里面，直接访问 Coding Pages 给的域名或使用自己绑定的域名就能看到博客了！

请不要忘记开通 Pages 服务，不然这里的信息一个都找不到
